# 《三角机构（Triangle Agency）》项目发展规划与复盘

## 一、 项目背景与核心愿景
本项目旨在打造一个专用于《三角机构（Triangle Agency）》桌面角色扮演游戏（TRPG）的**一站式在线辅助跑团与社区平台**。

通过结合现代企业官僚主义、超自然恐怖与办公室喜剧的世界观，平台试图满足玩家（外勤特工）与主持人（总经理 GM）的两大核心诉求：
1. **实时游戏控制台**：提供支持多实例高并发的 WebSocket 聊天室以进行实时文字跑团，并集成电子角色卡（ARC系统）辅助游戏内判定与管理。
2. **沉浸式异步社区（论坛跑团/情报站）**：打造一个高度贴合游戏背景的论坛系统（内部代称：冷冻酸奶茶水间），支持 Play-by-Post（文字团）和玩家间的情报、模组、经验交流。

## 二、 当前技术栈与架构
项目采用前后端分离的现代化 Web 应用架构，具备良好的分布式扩展能力。

### 1. 前端 (Frontend)
* **核心框架**：Vue 3 + Vite
* **UI 组件库**：Ant Design Vue
* **路由与状态**：Vue Router
* **文本渲染**：md-editor-v3 (用于发布和预览长篇情报与跑团安利)
* **网络请求**：Axios (集成统一的 JWT Bearer 拦截器)

### 2. 后端 (Backend)
* **核心框架**：Go 1.25 + GoFrame (gf v2)
* **数据库**：MySQL 8.0 (承载用户、角色、房间、帖子、评论等多态业务表)
* **实时通讯**：Gorilla WebSocket (维持跑团房间的长连接)
* **消息中间件**：RabbitMQ (支持聊天室多实例的 fanout 消息广播，具备本地内存降级机制)
* **对象存储**：MinIO (处理用户头像、跑团素材等静态资源)
* **认证鉴权**：golang-jwt/jwt/v5
* **缓存支持**：patrickmn/go-cache (本地内存缓存)

## 三、 当前开发进度盘点

### ✅ 已完成的核心基建与功能
1. **数据库层面的重构与优化**：
   - 整合了用户关注体系（`user_follows`）。
   - 统一了多态评论体系（`comments` 表支持帖子、动态等多场景复用）。
   - 优化了角色卡（`role_cards`）与用户（`users`）表，使用 JSON 字段（`extra_info`）收敛零散的设定与配置，移除了冗余字段。
2. **A/B 测试流量分发系统**：
   - 后端已集成轻量级的 A/B 测试中间件，基于用户标识（Hash 取模）稳定分配实验组与对照组。目前在论坛帖子列表接口已跑通验证。
3. **假数据生成脚手架 (Seeder)**：
   - 编写了 `cmd/seeder/main.go`，可通过 `gofakeit` 批量生成测试用户、关注关系、跑团角色卡、灌水帖子与多态评论，极大方便了本地开发与测试。
4. **房间与选角系统**：
   - 实现了房间的 CRUD，支持密码、私密性及游戏系统设定。
   - **核心突破**：实现了“先建卡、后入房”的机制。玩家可以在个人中心创建多张角色卡，在加入具体房间时选择指定的角色卡（包含特有的 QA 属性与 ARC 设定）进行绑定记录（`trpg_room_member`）。
5. **多人 WebSocket 聊天室基建**：
   - 跑团聊天中继站的核心已打通，支持加入/离开频道的生命周期管理。
   - 成功集成 RabbitMQ 进行跨实例消息广播。
6. **论坛互动体系强化（点赞Toggle机制）**：
   - 建立通用的点赞记录表（`user_likes`），支持防止重复刷赞的 Toggle 机制（点赞/取消点赞）。
   - 重构后端帖子与评论列表视图，批量注入当前登录用户的 `isLiked` 状态。
   - 包含完整的集成测试脚本 `test_likes.go` 用于自动化验证状态一致性。

## 四、 部署与运行指引
*(参考 `RUN.md` 与 `DEPLOY.md`)*

* **本地快速启动**：
  * 确保本地 MySQL 启动并拥有 `trangleagent` 数据库。
  * 后端：`cd Backend && go run main.go`
  * 前端：`cd Frontend && npm run dev`
* **Docker Compose 完整部署**：
  * 在项目根目录执行 `docker compose up -d --build`，将一键拉起包括 Nginx(前端)、Go(后端)、MySQL、MinIO、RabbitMQ 在内的所有容器矩阵。
